# 公众号内容打分

## ⚠️ 永久核心规则：5 层评分体系

**所有模式 B（全文打分）必须严格按 5 层体系执行，不可使用其他框架**。

| 层 | 维度 | 满分 | 评估什么 |
|---|---|---|---|
| ① | 标题流量分 | 100 | 读者会不会点开 |
| ① | 标题品牌分 | 100 | 像不像本人 / 会不会继续关注 |
| ② | 阅读推进力 | 20 | 每一节是否回答上一节留下的疑问 |
| ③ | 案例密度 | 20 | 案例 > 观点，还是观点 > 案例 |
| ④ | 认知升级度 | 30 | 是否完成多次升维（不是 X，是 Y）|
| ⑤ | 金句密度 | 30 | 是否有可截图传播的记忆点 |
| ⚠️ | PPT 惩罚 | -20 | 小标题是否像咨询报告（本质/方法论/启发/意味着）|

**4 个核心指数**（替代单一总分）：

| 指数 | 公式 | 满分 | 评级锚点 |
|---|---|---|---|
| 流量指数 | 标题流量分 | 100 | 80+ 高 / 65-79 中 / <65 弱 |
| 完读指数 | 阅读推进力 | 20 | 18+ 高 / 12-17 中 / <12 弱 |
| 转发指数 | 认知升级度 + 金句密度 | 60 | 50+ 高 / 35-49 中 / <35 弱 |
| 长期关注指数 | 标题品牌分 + 案例密度 | 120 | 100+ 高 / 80-99 中 / <80 弱 |

**判定等级**：
- ✅✅ 爆款（4 高）
- ✅ 强推（3 高 1 中）
- ⚠️ 流量撑场 / ⚠️ 浪费好内容 / ⚠️ 短板效应
- ❌ 不推荐

**PPT 惩罚触发词**（小标题层面）：本质 / 本质是什么 / 方法论 / 启发 / 意味着 / 总结 / 复盘 / 深度分析 / 思考框架 / 深度思考 / 三个步骤 / 五条建议
- 单次 -2
- 连续 2 次 -5
- 连续 3+ 次 -10

**绝对禁止**：
- ❌ 用 9 维 / 8 维 / 120 分制等其他旧框架
- ❌ 用单一总分替代 4 指数
- ❌ PPT 惩罚只算标题不检测正文里的"本质上/方法论上"
- ❌ 阅读推进力漏算小标题连接处

两种模式，按用户问的来：

| 模式 | 评估对象 | 何时用 | 框架 |
|---|---|---|---|
| **A. 标题+封面打分** | 只评标题+封面在信息流的吸引力 | 写完正文、做封面之前；封面做完、发布前；历史复盘 | 七维 100 分制（见下文） |
| **B. 全文打分** | 评整篇文章的传播力、阅读体验、认知价值 | 写完正文后看整体内容质量；发布前最终质检；写完一批文章对比 | 5 层体系（标题层 100+100 + 推进力 20 + 案例 20 + 认知 30 + 金句 30 + PPT 惩罚 -20）= 4 个核心指数 |

**关键原则**：
- 模式 A 不评正文（用户明确说不要）
- 模式 B 评全文但**不评观点对错**（用户明确说不要）——只评"作为公众号内容的传播力"
- 用户没说哪种时，默认按"标题+封面打分"（模式 A）

## 模式 A：标题 + 封面流量潜力打分

只评估**标题 + 封面**在公众号信息流里的吸引力，不评正文质量。

## 适用场景

- 写完文章准备发之前，给标题和封面打分
- 看别人文章时复盘为什么火/为什么没人看
- 同一篇文章尝试多个标题时挑最优

## 评分框架（双独立分）—— 流量分 100 + 品牌分 100

**核心原则**：评的是**两个不同问题**：
- **流量分**：读者会不会点开 + 转发动机 + 好奇心
- **品牌分**：读者会不会继续关注你 + 信不信你 + 是不是本人

**绝对禁止的隐性偏差**：
- ❌ 不要因为标题有"我以为/后来发现/帮我/时序"就自动给高分
- ❌ 不要因为标题结构完整（弧线、对比、时序）就给高分
- ❌ 不要把"抽象高级"误判成"高点击"
- ❌ 不要把"夸张统计"（92%/99%）误判成"高点击潜力"——读者天然不信，反而失分
- ❌ 不要把流量分和品牌分混着算（流量高的标题可能品牌分极低，反之亦然）
- ✅ 用**两张牌**思考：流量分 80+ 才考虑点开，品牌分 80+ 才考虑发布

### 双独立分对照

| 分数 | 评估对象 | 核心问题 | 包含维度 |
|---|---|---|---|
| **流量分 100** | CTR + 转发 + 好奇心 | 读者会不会点开？会转发吗？ | 七维流量潜力（代入/热词/1秒理解/冲突/信息承诺/传播/节奏）|
| **品牌分 100** | 信任感 + 一致性 + 长期形象 | 读者点了不失望？像本人会说的话吗？ | 真实性 + 信任成本 + 账号风格 + 一致性 |

### 用户举例（来自真实打分）

| 标题 | 流量分 | 品牌分 |
|---|---|---|
| Claude Code 半年，92% 都在做错 | 88 | 60 |
| 我以为 Claude Code 用来做产品，结果是做产品实验 | 82 | 92 |
| 大家都在 vibe coding，软件工程反而变重要了？ | 90 | 95 |

**关键洞察**：
- "92% 都在做错" **流量高（88）但品牌低（60）**——数字党拉低品牌分
- "我以为...结果做产品实验" **流量中（82）但品牌极高（92）**——亲历弧线拉高品牌分
- "大家都在...反而..." **流量高（90）品牌极高（95）**——观察式反问双赢
- **理想标题：双 80+**；**最佳：双 85+**
- **流量 80+ 但品牌 < 70 = 慎用**（短期流量自杀长期品牌）
- **品牌 80+ 但流量 < 70 = 浪费好内容**（好标题没发挥）
- **双低 = 直接放弃**

### 流量分 vs 品牌分 vs 老 140 分制

| 维度 | 老 140 分制 | 新双 100 分制 |
|---|---|---|
| 总分 | 140 | 两个 100 |
| 流量 | 七维 100 | 七维 100（不变）|
| 真实性 | 15 | 品牌分 100 的一部分 |
| 信任成本 | 15 | 品牌分 100 的一部分 |
| 账号风格 | 10 | 品牌分 100 的一部分 |
| 一致性 | Step 0.5 扣分 | 品牌分 100 的一部分 |
| **核心差异** | 单一总分，需要权衡 | **双独立分，各有阈值，直接判断** |

### 第一段：流量分（100 分）—— 七维流量潜力

**核心问题**：读者会不会点开？会转发吗？会产生好奇心吗？

**这是 CTR + 转发 + 好奇心三个子问题的综合评估**。

| 维度 | 分值 | 核心问题 | 评分锚点（举例） |
|---|---|---|---|
| 1. 目标读者代入感 | 20 | 读者一眼觉得"这是在说我"？ | 有明确身份词/场景词/工具词：程序员、Claude Code、Cursor、Codex、AI创业、产品经理、OpenClaw、vibe coding、Agent |
| 2. 热词与讨论场接入 | 15 | 是否接入目标读者最近正在讨论的话题？ | 是否包含最近 AI/产品/创业圈高频词（vibe coding、Claude Code、Cursor、Codex、Agent、Skill、MCP、AI创业），是否能让读者感觉"最近正在发生" |
| 3. 1秒理解成本 | 15 | 读者 0.3 秒能不能立刻知道在讲什么？ | 标题能不能在第一眼就传达"这篇文章解决什么"；是否需要读者思考两遍才懂；是否过于抽象（"淘汰错误想法"这种表达要判断是否清楚） |
| 4. 认知冲突/反直觉 | 20 | 是否有"我以为A，结果B"或"大家以为A，我看到B"？ | 反转/反问/反常识；是否能制造"为什么会这样"的好奇 |
| 5. 信息承诺清晰度 | 15 | 点进去能获得什么是否明确？ | 是经验、复盘、判断、方法论，还是泛泛观点；是否避免空泛大词（"AI时代""未来趋势"） |
| 6. 传播/讨论欲 | 10 | 是否容易让人想转给同事/朋友？ | 能否引发认同/争论/站队/"这说的是我"；有无金句潜力 |
| 7. 语言锋利度与节奏 | 5 | 是否顺口、短、利，有记忆点？ | 是否避免太长太绕；有无"动词钩子"（淘汰/做对/重做）；有无对仗/押韵 |

**注**：封面视觉焦点、标题封面一致性 → 移到 Step 0 单独预检（封面打分），不挤进标题七维。

### 第二段：品牌分（100 分）—— 真实性 + 信任成本 + 账号风格 + 一致性

**核心问题**：读者点了不失望吗？像本人会说的话吗？信不信？

**这是信任感 + 一致性 + 长期形象 + 真实性承诺四个子问题的综合评估**。

#### 品牌分四子项

| 子项 | 分值 | 评估问题 |
|---|---|---|
| 真实性与预期一致性 | 30 | 标题承诺的内容，正文是否真的提供？数字是否有真实来源？ |
| 信任成本（Trust Cost） | 30 | 读者是否天然相信？还是触发怀疑（"92% 怎么算的？"）？ |
| 账号风格匹配度 | 25 | 是否符合作者长期品牌？像本人会说的话吗？ |
| 标题-正文一致性 | 15 | Step 0.5 一致性扣分整合（标题视角是否与正文一致）|
| **品牌分小计** | **100** | |

#### 品牌分 vs 流量分

- 流量分高 + 品牌分低 = **流量党/数字党/营销号**（短期流量，长期掉粉）
- 流量分低 + 品牌分高 = **好内容没好标题**（浪费）
- 流量分高 + 品牌分高 = **真正的好标题**（CTR + 复购 + 转发 + 关注）

#### 子项 1：真实性与预期一致性（30 分）

**这是"防止短期流量自杀长期品牌"的核心维度**——读者点开是第一步，读完不失望并继续关注是第二步。

**评估清单**：

1. 标题承诺的内容，正文是否真的提供
2. 标题中的数字是否有真实来源（正文数据 / 作者亲历 / 行业可查）
3. 标题是否会让读者形成错误预期
4. 标题是否存在"点击率高但读完失望"的风险

**加分项**：
- + 标题中的数字来自作者真实经验
- + 标题中的数字来自正文明确数据
- + 标题与正文高度一致
- + 读者读完不会产生"被骗进来"的感觉

**扣分项**：
- - 数字看起来像行业统计，但正文无法支撑（如"90% 程序员都不知道"，正文没有数据来源）
- - 标题承诺的内容正文没有回答
- - 为了点击率故意放大结论
- - 读者点进来期待 A，结果正文讲 B

**评分标准**：

| 分数 | 含义 | 典型症状 |
|---|---|---|
| 26-30 | 标题与正文高度一致，无明显误导 | 数字有来源 / 承诺的具体 / 读完符合预期 |
| 18-25 | 存在轻微包装，但整体符合预期 | 标题有"放大"但正文撑得住 |
| 10-17 | 存在明显预期偏差 | 数字悬空 / 承诺的内容只在边缘提到 |
| 0-9 | 严重标题党风险 | 数字编造 / 内容与承诺完全不符 |

#### 子项 2：信任成本（Trust Cost，30 分）

**这是"读者天然信不信"的维度**——和"真实性"互补但不同：
- **真实性**：标题-正文是否一致（事后检查）
- **信任成本**：读者点开前是否天然相信（事前判断）

**核心问题**：标题里的信息，读者第一眼读过去，**是否天然相信**？还是心里冒问号（"92% 怎么算的？" / "凭什么这么说？"）？

### 信任成本锚点

| 信任等级 | 典型表现 | 读者心理 |
|---|---|---|
| **极高信任（26-30）** | "大家都在 vibe coding，软件工程反而变重要了？" / "AI 写的代码越来越多，调试为什么反而变难？" | 读者觉得："可以聊聊" / "听着有道理" / "我也想说这个" |
| **高信任（20-25）** | "我以为 Claude Code 用来做产品，结果是做产品实验" / "Claude Code 用半年我才发现：它做的是产品实验" | 读者觉得："这是作者经验" / "我信他在做这件事" |
| **中等信任（13-19）** | "Claude Code 半年，11% 才是做对" | 读者觉得："11% 哪里来的？看看怎么说的" |
| **低信任（6-12）** | "用 Claude Code 做产品，92% 都在做错" | 读者觉得："92% 怎么算的？" / "凭什么是 92% 不是 80% 或 95%？" |
| **极低信任（0-5）** | "99% 程序员都理解错了 AI" / "X% 的人都不知道的 Y 个秘密" | 读者觉得："典型营销号" / "直接划走" |

### 信任成本评分标准

| 分数 | 含义 | 典型症状 |
|---|---|---|
| 26-30 | 极高信任，读者天然相信 | 观察式/反问式/作者亲历式 |
| 20-25 | 高信任，读者有理由信 | 完整弧线 + 真实经验暗示 |
| 13-19 | 中等信任，读者想验证 | 有具体数字但缺少来源或上下文 |
| 6-12 | 低信任，读者起疑 | 数字党 / 百分比 / "X% 的人 Y" |
| 0-5 | 极低信任，读者直接划走 | 99%/90%/典型营销号句式 |

### 信任成本加分项

- + 观察式/反问式（"大家都在 X，反而 Y"）——读者觉得是观察不是吹牛
- + 作者亲历式（"我以为...结果..."）——读者觉得是真实经验
- + "反而/原来/其实" 这类转折词——读者觉得是发现不是宣告
- + 没有具体数字 + 有具体场景——读者不觉得被统计

### 信任成本扣分项

- - **统计式数字**（"92%"/"99%"/"X% 的人"）——读者会想"数据从哪来"
- - **百分比** + **强判断**（"92% 都在做错"）——双重失分
- - **极端数字**（"99%"/"100%")——典型营销号
- - **"X% 的人都不知道"**——典型震惊体

### 信任成本 vs 真实性

**两个维度独立判断，不要混用**：

| 情况 | 真实性 | 信任成本 |
|---|---|---|
| "92% 都在做错" + 正文有 92% 数据 | 高（14/15） | **低（4-6/15）**——读者天然起疑 |
| "92% 都在做错" + 正文无 92% 数据 | 低（6/15） | **低（4-6/15）**——双重失分 |
| "我以为...结果..." + 正文有支撑 | 高（14/15） | **高（12-13/15）**——读者天然信 |
| "大家都在 X，反而 Y" + 正文有支撑 | 高（14/15） | **极高（14-15/15）**——观察式天然信 |

**核心洞察**：信任成本更多由**标题句式**决定，不一定由数字真假决定。**反问/观察/转折**天然比**陈述/数字/宣告**可信。

### 信任成本与"读者群"耦合

**同样的标题，在不同读者群里信任成本不同**：

- **AI/产品/创业号读者**：更信"观察/亲历/弧线"，更不信"统计数字"
- **大众/情感/财经号读者**：相对能接受"统计数字"（因为习惯了）
- **专业/技术号读者**：能接受具体数字（因为可以查证），但不接受"百分比+强判断"

**作者所在读者群**：AI 实践 / OpenClaw / Claude Code / 产品实验 / 软件工程 / 系统思维 / 反馈循环 / 复杂性管理

→ **作者读者群天然对"百分比+强判断"低信任**——"92% 都在做错"会触发"92% 怎么算的？"

#### 子项 3：账号风格匹配度（25 分）

**这是"防止短期流量破坏长期品牌"的核心维度**——某些标题短期能涨粉 1000，但风格不符会让老读者觉得"号变了"，长期掉粉更严重。

**作者特点**（必须匹配）：
- AI 实践
- OpenClaw
- Claude Code
- 产品实验
- 软件工程
- 系统思维
- 反馈循环
- 复杂性管理

**作者不是**（必须避免）：
- 情绪博主
- 焦虑博主
- 数字党
- 标题党

**加分项**：
- + 真实经验
- + 方法论提炼
- + 反直觉观察
- + 第一手实践
- + 系统思维暗示（"反馈循环""复杂性"）

**扣分项**：
- - 过度夸张
- - 强情绪刺激
- - 制造焦虑
- - 数据党标题（"90% 的人都不知道"）
- - 震惊体
- - 鸡汤化

**评分标准**：

| 分数 | 含义 | 典型症状 |
|---|---|---|
| 22-25 | 非常符合账号长期品牌 | 真实经验 / 方法论 / 第一手实践 |
| 14-21 | 短期流量可能好，但不利于品牌积累 | 略夸张 / 略情绪化 |
| 0-13 | 明显偏离账号定位 | 数字党 / 震惊体 / 焦虑 / 标题党 |

## 扣分项（独立于七维，可叠加）

**重要：扣分项要严于加分项**——很多标题"没扣分但也没真的给读者留下印象"，这种其实是低潜力。

| 扣分症状 | 扣分 | 典型案例 |
|---|---|---|
| **直接给结论但没有问题感** | -5 ~ -15 | "AI 时代产品方法论彻底变了"（直接宣布观点）|
| **抽象词太多，缺少具体对象** | -5 ~ -20 | "做产品实验"（"实验"太抽象）/ "AI 时代的本质"（"本质"空）|
| **太像行业评论/AI 泛观点** | -5 ~ -15 | "我看到了 AI 创业的真相" / "未来已来" |
| **读者无法判断和自己有什么关系** | -10 ~ -20 | "我们都在被 AI 重塑"（"我们"太宽泛）|
| **虽然结构完整但点击欲弱** | -5 ~ -15 | 完整"我以为A结果B"弧线但缺少热词/身份词 |
| **字数 > 28 且信息密度不够** | -5 ~ -10 | 32 字的口语化标题（前 22 字铺垫，钩子被截断）|
| **关键词太宽泛，没有具体锚点** | -5 ~ -15 | "AI 时代""未来趋势""大模型""创业"（缺具体工具/场景）|

## 工作流

### Step 0：标题长度预检（必走，独立于七维）

公众号信息流里，标题的物理显示宽度有限。
- 手机端单行约显示 15-20 字（中文按 1 字算，英文按 1 字符算 1 字，数字按 1 字算）
- 超过 22 字必被截断，钩子失效
- 短于 11 字，身份词/反转/冲突塞不进去
- **15-20 字是最佳区间**（既能在缩略图完整显示，又能塞下"身份词 + 反转"）

**字数扣分表**（从总分里直接扣，不挤进七维）：

| 标题字数 | 扣分 | 原因 |
|---|---|---|
| 15-20 字 | 0 | 最佳区间，单行完整显示+信息密度够 |
| 13-14 或 21-22 | -1 | 边界外，可接受但欠优 |
| 11-12 或 23-24 | -2 | 明显偏离，信息密度或显示完整度不足 |
| 9-10 或 25-26 | -3 | 严重偏离，要么太空要么必截断 |
| <9 或 ≥27 | -4 | 极短（丢失冲突/身份词）或必截断（前缀钩子丢失） |

**注**：
- "字" = 中文字符、英文字母按 1 字符、数字按 1 字符、英文空格不计入
- 例外：纯英文标题（无中文）按 1 词 = 1.5 字折算

### Step 0.5：标题-正文一致性预检（必走，独立于七维）

**这是公众号最隐蔽也最致命的扣分项**——读完落差让读者觉得"被骗了"，直接打击转发欲（往往是低传播率的根因）。

**判断方法**：
1. 标题里的核心关键词（如 "Claude Code"）点出来的是什么视角？—— 工具视角 / 方法论视角 / 趋势视角
2. 正文实际在讲什么视角？—— 用一句话总结正文的真正核心观点
3. **两者是否同一视角？**

**扣分表**（从总分里直接扣，不挤进七维）：

| 错位程度 | 扣分 | 典型症状 |
|---|---|---|
| 完全一致 | 0 | 标题承诺的"会得到什么"和正文实际给的"是什么"对得上 |
| 视角错位 | -3 | 标题说"工具"，正文讲"方法论"；或标题说"个人经验"，正文讲"行业判断"（典型：工程师点进"Claude Code 半年"标题，结果读到产品哲学） |
| 范围错位 | -5 | 标题限定一个具体场景（如"做八字 Skill"），正文泛化到整个产品方法论（读者期望八卦/具体复盘，结果读到一篇大而泛的方法论文） |
| 承诺完全不符 | -8 | 标题说"A 体验"，正文讲"B 主题"（这种基本是不小心的写作事故） |

**典型错位举例**（要记住，下次自动避坑）：

❌ 标题："Claude Code 用半年我才发现：它做的是产品实验"
✅ 正文实际：讲的是 AI 时代产品方法论（"做对→做快"）
❌ 错位：标题锁定工具视角（"它做的是 X"），正文实际是方法论视角
❌ 后果：工程师点进来看 Claude Code 技巧 → 读完发现是产品哲学 → 觉得"被骗了" → 不转发
**应改成**：
- 改成方法论视角："Claude Code 用半年我才发现：做产品方法论变了" / "Claude Code 半年后：AI 产品方法论从'做对'变成'做快'" / "Claude Code 用半年：让我重新理解'做对'和'做快'"
- 或者改成工具视角的真正具体经验（不写方法论）

**修法检查清单**：
- 改完标题后，把标题关键词列出来
- 再用一句话总结正文核心观点
- 两者视角必须一致（要么都"工具"，要么都"方法论"，要么都"个人经验"）
- 不能一个限定"工具"另一个讲"方法论"

### Step 0.6：标题拗口度预检（必走，独立于七维）

**拗口 = 重复读 = 信息流划走**。公众号信息流里读者读标题 0.3 秒，**任何需要"重新组句才能懂"的标题就是拗口**。

**判断方法（逐条检查）**：

1. **双引号/书名号数量**：标题里出现 2 个及以上"X"或《X》→ 大概率拗口（每出现一组 +1 拗口分）
2. **主语跳跃**：标题前半部分在讲 A，后半部分突然转 B，中间没有连接词 → 拗口
3. **"X 不如 Y" 结构**："X"和"Y"必须都是读者已知的明确概念，否则读者会卡在"X是什么"——拗口
4. **嵌套结构**："A，B，C，D"超过 3 个短句、或"X，Y"出现 2 次以上 → 拗口
5. **绕口令测试**：标题一口气读完是否要中途换气 2 次以上？要换气 2 次 = 拗口

**拗口扣分表**（从总分里直接扣，可叠加，最多 -8）：

| 拗口症状 | 扣分 | 典型案例 |
|---|---|---|
| 0 个拗口症状 | 0 | 流畅，一口气读完 |
| 1 个轻微症状（如 1 对引号 / 1 次主语跳跃） | -2 | 还能读，但不顺 |
| 2 个症状（如 2 对引号 + 1 次主语跳跃） | -4 | 明显拗口 |
| 3 个症状（如 2 对引号 + 主语跳跃 + "X 不如 Y" 都陌生） | -6 | 必须重读 |
| 4 个及以上 | -8 | 灾难级拗口 |

**改法原则**：
- **去掉引号**：除非你引的是大众熟悉的金句（"996" "35 岁"），否则一律去掉
- **主语统一**：标题全程围绕同一个主语，不要从 A 跳到 B
- **"X 不如 Y" 的 X 和 Y 必须是大众词**：X=快、便宜、稳（已知概念）；Y=慢、贵、玄（已知概念）。如果 X 或 Y 是生造词，必须改写成大众词
- **短句重排**：如果标题有 3 个以上短句，砍到 2 个短句（逗号最多 1 个）

**改法举例**：

❌ `Claude Code 半年，我才看懂"做对"不如"做快"`
拗口：2 对引号 + 主语跳跃（Claude Code → 做对 → 做快）+ "X 不如 Y" 双方都是生造短语
✅ 改成 `Claude Code 半年，我才看懂一件事：做对不如做快`（去掉引号，主语统一为"一件事"，X/Y 简化为大众词）
✅ 或 `Claude Code 半年，让我重新理解了一件事`（彻底规避"X 不如 Y"结构）
✅ 或 `Claude Code 半年，我才看懂"做对"是个错觉`（保留 1 对引号，但 X 是大众词，Y=错觉也是大众词）

### Step 1：接收用户输入

用户需要提供：
- 【目标读者】（必填，例：程序员 / AI工程师 / OpenClaw用户 / AI创业者）
- 【标题】（必填，**可以同时粘贴多个标题对比**）
- 【封面】（必填，图片路径或URL；如果只评标题不评封面，封面那部分标 N/A）
- 【已知参考数据】（可选，历史阅读/点赞/分享数据）
- 【文章核心观点】（强烈建议填——评分时需要用核心观点判断"信息承诺"是否兑现）

如果只提供标题没有封面：明确告知"封面维度无法评估"，其余六维照打。
如果提供多个标题：分别打分，最后输出"标题选择建议"（按真实点击潜力排序）。

### Step 3：标题七维打分

每个维度**必须给具体扣分理由**，不能直接给总分。

#### 1. 目标读者代入感（/20）

**加分词**（命中任意+3~5分）：程序员 / 软件工程 / vibe coding / Claude Code / Cursor / Codex / Agent / OpenClaw / AI创业 / 副业 / 35岁 / 转型 / 失业 / 产品经理
**减分信号**：
- 标题里完全没有身份词/场景词（"我们""你"等代词不算）
- 标题用了"我们"这种宽泛代词，没有具体身份锚点

#### 2. 热词与讨论场接入（/15）

**加分**：标题包含最近3个月目标读者高频讨论的关键词（Claude Code / Cursor / Codex / Agent / Vibe Coding / Prompt / 上下文工程 / Skills / MCP / OpenClaw 等）
**减分**：
- 完全泛AI观点（"AI时代来临"），无具体产品/技术/事件挂钩
- 关键词太宽泛（"大模型""创业""未来"），无具体锚点（-5~-15）

#### 3. 1秒理解成本（/15）

**高分锚点**：
- 标题第一眼就能传达"这篇文章在解决什么"
- 动词具体（淘汰/重做/做错/验证）
- 抽象词少（避免"本质""价值""范式""复杂性"等空词）

**低分信号**：
- 需要思考两遍才懂
- 抽象词堆砌（"AI时代的本质"）
- 主语跳跃（读者要重新组句）

#### 4. 认知冲突/反直觉（/20）

**高分量表达**：
- "我以为A，结果B"（"以为…其实…"）
- 反问句（"为什么…？"）
- 反转（"不是X，是Y"）
- 数字反差（"92%技术评估准确率，只有11%需求命中"）
- 极值词（"最危险的""真正稀缺"）
- 强动词（淘汰/重做/颠覆）

**减分**：标题是"AI时代要X""程序员应该Y"这种无冲突的判断句

#### 5. 信息承诺清晰度（/15）

**加分**：读者能预知文章会给出"具体复盘/具体判断/具体数字"
**减分**：
- 标题像口号（"未来已来""新范式"）
- 标题太抽象（"复杂性""价值"等抽象名词堆砌）
- 标题没有承诺任何"读者会得到的东西"

**注**：如果用户提供了【文章核心观点】，要交叉验证——标题承诺的"会得到什么"和文章实际给的"是什么"是否对得上。

#### 6. 传播/讨论欲（/10）

- 标题有"站队空间"（"最危险的""真正稀缺"）：+2
- 标题带具体可转述结论（数字/对比/反常识）：+3
- 标题触达痛点但给出方向（"以后怎么办"）：+1
- 标题让读者想"这说的是我"：+2
- 标题让读者想转给同事/朋友：+2

#### 7. 语言锋利度与节奏（/5）

- 有强动词钩子（淘汰/做对/重做/颠覆/做错）：+2
- 对仗工整（4字 vs 4字，5字 vs 5字）：+1
- 押韵/金句潜力：+1
- 短句利落（无"在""的"等冗余）：+1

## 模式 B：全文打分（资深主编版）

**评整篇文章的传播力、阅读体验、认知价值**——不评观点对错，只评"作为公众号内容"的表现。

### 适用场景

- 写完正文后看整体内容质量
- 发布前最终质检（封面做完之后、模式 A 之外补一次"内容本体"评估）
- 写完一批文章横向对比哪些能爆
- 复盘旧文为什么没火

### 评分框架（5 层体系 + 1 个惩罚项）

**核心思想**：公众号文章不是咨询报告，是叙事。
- **标题层**评"会不会点开"（双独立分）
- **正文层**评"会不会读完 + 会不会转发 + 会不会关注"
- **PPT 惩罚**评"像不像咨询报告 vs 公众号叙事"

| 层 | 维度 | 分值 | 核心问题 |
|---|---|---|---|
| ① | **标题流量分** | 100 | 读者会不会点开？（CTR + 转发 + 好奇心）|
| ① | **标题品牌分** | 100 | 像不像本人？读者会不会继续关注？|
| ② | **阅读推进力** | 20 | 每一节是否回答上一节留下的问题？|
| ③ | **案例密度** | 20 | 案例 > 观点？还是观点 > 案例？|
| ④ | **认知升级度** | 30 | 读者是否获得"原来如此"？是否完成多次升维？|
| ⑤ | **金句密度** | 30 | 是否有可截图传播的记忆点？|
| ⚠️ | **PPT 惩罚** | -20 | 标题是否像咨询报告（"本质是什么/方法论/启发"）？|

**总分计算**：
- 标题层总分 = 流量分 + 品牌分（独立呈现，不合并）
- 正文层总分 = 阅读推进力 + 案例密度 + 认知升级度 + 金句密度 + PPT 惩罚 = 100-20=80
- **4 个核心指数**（输出给用户）：
  - 流量指数 = 标题流量分（100 制）
  - 完读指数 = 阅读推进力（20 制）
  - 转发指数 = 认知升级度 + 金句密度（60 制）
  - 长期关注指数 = 标题品牌分 + 案例密度（120 制）

### 工作流

**Step 1：接收用户输入**

用户需要提供：
- 【正文】（必填，markdown 全文）
- 【标题】（必填，文章标题）
- 【目标读者】（可选，但强烈建议填——影响认知增量、案例质量的判断锚点）

**Step 2：5 层体系打分**

每个维度**必须给具体扣分理由**。

#### ① 标题层（流量分 100 + 品牌分 100）

**这一层沿用模式 A 的双独立分**（七维 + 真实性 + 信任成本 + 风格 + 一致性），详细评分框架见模式 A。

**Step 2.1：流量分（/100）**
- 七维流量潜力：代入/热词/1秒理解/冲突/信息承诺/传播/节奏
- 决定"读者会不会点开"

**Step 2.2：品牌分（/100）**
- 真实性 30 + 信任成本 30 + 账号风格 25 + 标题-正文一致性 15
- 决定"读者会不会继续关注"

**字数扣分**：标题长度按模式 A 的 Step 0 扣分表处理，**不挤进此维度**——见总分前的"零、标题长度预检"。

---

#### ② 阅读推进力（/20）

**核心问题**：每个小标题之间，**是否形成"问-答"链条**？
- 读者看完第 N 节，最自然的疑问/期待是什么？
- 第 N+1 节是否精准回答了这个问题？
- 如果不回答，读者是不是会卡住/划走？

**评估方法**（逐节对照）：

对每两个相邻小标题：

```
❸ 上一节标题：[A 节标题]
↓
读者看完 A 后最自然会问什么？
↓ (评估下一节标题是否回答)
❹ 下一节标题：[B 节标题]
↓
B 是否回答了 A 留下的疑问？
```

**评分锚点**：

| 推进质量 | 表现 | 每节得分 | 累计换算 |
|---|---|---|---|
| **完美推进** | 下一节标题直接回答上一节疑问 | 4-5 | 18-20 |
| **自然推进** | 下一节方向对，不直接回答 | 3-4 | 12-17 |
| **硬切** | 下一节和上一节有逻辑断点 | 2-3 | 8-11 |
| **断层** | 下一节读者预料之外 | 0-2 | 0-7 |

**评分算法**：

```
阅读推进力得分 = Σ(每个连接处得分) / (N-1) × 20
```

或者更简单：
- N-1 个连接处总分 = X
- 阅读推进力得分 = X / (5×(N-1)) × 20

**实战例子**（v19a 文章，迭代后）：

| 连接 | 上一节 | 下一节 | 推进质量 | 得分 |
|---|---|---|---|---|
| ❶→❷ | AI 时代最大的红利是「开发变快」 | 但很多 Demo 根本没人用 | 完美：反差 | 5 |
| **❷→❸** | **但很多 Demo 根本没人用** | **后来我发现，我的问题根本不在 Demo** | **完美：直接回答** | **5** |
| ❸→❹ | 后来我发现，我的问题根本不在 Demo | 以前做错一个产品假设要付出多少代价 | 完美：解释代价 | 5 |
| ❹→❺ | 以前代价 | AI 时代这道题的成本下降了两个数量级 | 完美：反转 | 5 |
| ❺→❻ | AI 时代成本下降 | 怎么用这个视角看 Demo | 自然：方法论过渡 | 4 |
| ❻→❻.5 | 怎么用这个视角看 Demo | 这件事的本质是什么 | 完美：升维 | 5 |
| ❻.5→❼ | 这件事的本质 | 这件事对你意味着什么 | 完美：回到读者 | 5 |

**7 个连接处总分 34/35，推进力得分 = 34/35 × 20 = 19.4**

**读者自然疑问 vs 标题匹配的关键模式**：

| 上一节留下的疑问 | 下一节应该长这样 |
|---|---|
| 然后呢？ | "我发现..." / "后来..." / 故事展开 |
| 怎么做到的？ | 拆解步骤 / 现场感 |
| 为什么？ | 解释原因 / 揭示动机 |
| 真的吗？ | 案例 / 数据 / 反例 |
| 跟我有什么关系？ | "对你" / "我们" / 具体场景 |

**扣分信号**：
- ❌ 上一节留强疑问，下一节用"我们先说说另一个话题"开始
- ❌ 小标题之间出现"主题漂移"（A 讲产品 B 讲职场）

**加分信号**：
- ✅ 上一节留强疑问，下一节第一句就回答
- ✅ 上一节是"问题"，下一节是"答案"——完美的问-答链条

**为什么"后来我发现，我的问题根本不在 Demo"是完美推进**：
- 读者看完❷ 会问："为什么没人用？"
- 下一节标题直接说"我的问题根本不在 Demo"——直接回答"不是 Demo 的问题，是我的问题"
- 读者立刻想"那你的问题是什么？"——顺滑进入八字故事

#### ③ 案例密度（/20）

**核心问题**：案例 > 观点，还是观点 > 案例？
- **观点 > 案例** = 咨询报告感（"提出一个观点，举一个例子说明"）❌
- **案例 > 观点** = 公众号叙事感（"先讲一个故事，故事里冒出观点"）✅

**评分锚点**：

| 案例密度 | 表现 | 分数 |
|---|---|---|
| **案例 > 观点（18-20）** | 至少 2 个真实案例，案例推动观点，案例先于观点出现 | 八字 skill、健身 skill 两个完整故事 → 推出"产品实验" |
| **案例 = 观点（12-17）** | 案例只有一个，或案例和观点并列 | 单一案例 + 多观点 |
| **观点 > 案例（0-11）** | 纯观点/纯方法论，案例只是装饰 | "3 个步骤教你做对" |

**关键判断**：
- 真实案例 ≠ 假设举例（"假设你是一个产品经理"不叫案例）
- 真实案例 = 有产品名 + 有时间 + 有用户行为 + 有反转
- 案例必须**独立成故事**（200 字以上有画面），不是一句话带过

**实战例子**（v19a 文章）：
- 案例 1：八字 Skill（300 字故事，反转：八字不是产品，是入口）
- 案例 2：健身 Skill（250 字故事，反转：记录不是产品，决策才是）
- 两个案例**都先于观点出现** → 推出"产品实验"方法论
- **得分：20/20**（满分）

**为什么"案例密度"是 5 层中最易得高分的**：
- 大部分 AI 公众号都在做观点 > 案例（输出方法论）
- 你只要做到**真实案例 + 故事化 + 案例先于观点**，就自动超过 80% 的对手
- 这是你**最近文章最大的优势**——八字 + 健身两个完整故事

**扣分信号**：
- ❌ 全文只有观点没有案例 → 0-5
- ❌ 案例 = "假设你..." 或 "我朋友..."（无具体产品名/数据）
- ❌ 案例只是观点的注脚（"比如 X"一句话带过）

#### ④ 认知升级度（/30）

**核心问题**：读者是否获得"原来如此"？是否完成**多次升维**？

**升维的定义**：
- 第 0 维：现象（"我做了 X"）
- 第 1 维：解释（"X 是 Y"）
- 第 2 维：升维（"X 不是 Y，是 Z"）
- 第 3 维：再升维（"不是 Z 的事，是 W 的事"）

**评分锚点**：

| 升维次数 | 表现 | 分数 |
|---|---|---|
| **出现二次升维（27-30）** | 读者完成"现象→解释→升维→再升维"完整链条 | 这篇：产品开发 → 产品实验 → 做对 → 做快（4 个层级）|
| **出现一次升维（20-26）** | 读者完成"现象→解释→升维"三层 | 软件工程那篇：代码复杂性 → 系统行为复杂性（3 个层级）|
| **没有升级（0-19）** | 全文止步于现象或解释 | "我做了 X，验证出 Y" 没有升到方法论/哲学层 |

**实战例子**（v19a 文章）：

| 维度层级 | 内容 |
|---|---|
| 现象 | 做了 Demo，没人用 |
| 解释 | 不是 Demo 的问题，是我的问题 |
| 升维 | AI Demo 真正的价值不是上线，是验证 |
| **再升维** | AI 时代方法论从"做对"变成"做快" |
| **再升维** | 不是"做快"本身，是"从失败里挖认知" |

**5 个层级，2 次明确升维 → 得分 28/30**

**为什么"再升维"是 27-30 分的关键**：
- 一次升维 = 优秀公众号水平（80% 头部）
- 二次升维 = 顶级公众号水平（前 5%）
- 没有升维 = 知乎/即刻水平（虽然有观点但不升级）

**升维触发词**：
- "不是 X，是 Y" → 升维 1
- "不是 Y 的事，是 Z 的事" → 升维 2
- "你以为 X，其实 Y，再深一层是 Z" → 升维 3

**扣分信号**：
- ❌ 全文止步于"我做了 X → 验证出 Y" → 0-10
- ❌ 一次升维后再无深化 → 11-20
- ❌ 升维是"硬拗"的（缺乏案例/数据支撑）→ 升维分再高也不算

---

#### ⑤ 金句密度（/30）

**核心问题**：是否有**可截图传播**的记忆点？

**为什么这个维度扩到 30 分**：
- 公众号文章的传播单位 = 金句（一句话截图）
- 没有金句 = 没有转发燃料
- 你最近文章**有观点但没记忆点**——这是新增这个维度的最大动机

**评分锚点**：

| 金句数量 | 表现 | 分数 |
|---|---|---|
| **2 个以上可截图传播句（25-30）** | 多句"截图发朋友圈不丢脸"的金句 | 2-3 个金句分散在不同章节 |
| **1 个金句（15-24）** | 有一句"可传播"的话 | 1 个金句 + 其他是普通表达 |
| **没有金句（0-14）** | 全文都是叙述和论证 | 0 个金句 |

**金句的判断标准**：
- ✅ 反直觉 + 短句 + 有视觉感
- ✅ 单独成段也成立
- ✅ 截图发朋友圈不丢脸
- ✅ 出现"原来如此"或"那为什么..." 的反应

**实战例子**（v19a 文章的金句）：

| 金句 | 位置 | 评估 |
|---|---|---|
| "AI 最大价值，可能不是把产品做出来。是让错误假设更快死掉。" | ❺ 节末 | 强金句（反直觉 + 短句 + 视觉感）|
| "AI 时代的产品方法论，从'做对'变成了'做快'。" | ❻.5 节末 | 强金句（对比 + 短句）|
| "不要问'这个 Demo 能不能上线'。要问'这个 Demo 让你搞懂了什么'。" | ❼ 节末 | 强金句（句式 + 行动指导）|

**3 个金句 → 得分 26/30**

**为什么"也"作为评分单位**：
- 1 个金句 = 读者会截图但不一定转发
- 2-3 个金句 = 读者会转发 + 朋友圈评论
- 0 个金句 = 读者读完觉得"有道理"但不传播

**扣分信号**：
- ❌ 全文都是叙述和论证，没有"可独立成段"的金句
- ❌ 金句 = "有道理的话"（"AI 时代最重要的是学习能力" 这种废话金句不算）

---

#### ⚠️ PPT 惩罚（-20）

**核心问题**：标题/小标题是否像**咨询报告标题**（"本质是什么""方法论""启发"）？

**这是新增的"叙事 vs 报告"分水岭**——
- **公众号 = 叙事**（"我做了 X，发现 Y"）
- **咨询报告 = 结构化**（"X 的本质是 Y"）

**触发惩罚的标题关键词**：

| 关键词 | 每次扣分 |
|---|---|
| 本质是什么 | -2 |
| 方法论 | -2 |
| 启发 | -2 |
| 这件事意味着什么 | -2 |
| 总结 | -2 |
| 复盘 | -2 |
| 深度分析 | -2 |
| 思考框架 | -2 |
| 深度思考 | -2 |
| 三个步骤 | -2 |
| 五条建议 | -2 |

**连续出现加倍**：
- 单次出现：-2
- 连续 2 次出现：-5
- 连续 3+ 次出现：-10

**为什么这是大惩罚（-20）**：
- 公众号不是报告，是叙事
- 读者看到"本质是什么"会下意识觉得"又是咨询文"
- 一次 PPT 标题 = 1 次劝退
- 连续 PPT 标题 = 整体调性崩

**实战例子**（v19a 文章的 PPT 检测）：

| 位置 | 标题 | 是否触发 |
|---|---|---|
| ❹ 节 | 以前做错一个产品假设要付出多少代价 | ✅ 否（叙事感）|
| ❺ 节 | AI 时代，这道题的成本下降了两个数量级 | ✅ 否（叙事感）|
| ❻ 节 | 怎么用这个视角看 Demo | ✅ 否（"怎么用"是行动）|
| **❻.5 节** | **这件事的本质是什么** | **❌ 是（触发 -2）** |
| ❼ 节 | 这件事对你意味着什么 | ⚠️ 边缘（"意味着"是软触发，-1）|

**触发 -3，连续扣 -5 → 总计 -6**

**实战得分**（v19a）：**PPT 惩罚 -6**

**修复建议**：
- "这件事的本质是什么" → "AI 时代，产品方法论被反转了"
- "这件事对你意味着什么" → "你可能不是在失败，是在做产品实验"

**为什么"意味着"是边缘触发**：
- "意味着"比"本质/方法论/启发"软
- 不强制扣分，但建议避免
- 改为"你可能..." / "这件事..." 更叙事

**特别说明**：PPT 惩罚只针对**小标题**，不针对正文里的"本质上/方法论上"等表达。

### Step 3：4 个核心指数评级

**5 层体系不再用单一总分，改用 4 个核心指数分别评级**：

| 指数 | 满分 | 评级锚点 |
|---|---|---|
| **流量指数** = 标题流量分 | 100 | 80+ 高 / 65-79 中 / <65 弱 |
| **完读指数** = 阅读推进力 | 20 | 18+ 高 / 12-17 中 / <12 弱 |
| **转发指数** = 认知升级度 + 金句密度 | 60 | 50+ 高 / 35-49 中 / <35 弱 |
| **长期关注指数** = 标题品牌分 + 案例密度 | 120 | 100+ 高 / 80-99 中 / <80 弱 |

**4 个指数判断逻辑**（**这是新体系的核心校准**）：

| 流量 | 完读 | 转发 | 长期关注 | 综合判定 |
|---|---|---|---|---|
| 高 | 高 | 高 | 高 | ✅✅ 爆款（4 高）|
| 高 | 高 | 高 | 中 | ✅ 强推（3 高 1 中）|
| 高 | 中 | 高 | 高 | ✅ 强推（3 高 1 中）|
| 高 | 中 | 中 | 中 | ⚠️ 流量撑场 |
| 中 | 高 | 高 | 高 | ⚠️ 好内容没好标题 |
| 高 | 中 | 中 | 低 | ❌ 流量党，自杀品牌 |
| 高 | 低 | 高 | 高 | ❌ 推进力崩，读者划走 |
| 中 | 中 | 中 | 中 | ❌ 平庸，无记忆点 |
| 任何一低 | | | | ⚠️ 短板效应 |
| 双低 | | | | ❌ 不推荐 |

**实战示例**（v19a 文章）：

| 维度 | 分数 | 评级 |
|---|---|---|
| 流量指数（标题流量分）| 96/100 | 高 |
| 完读指数（阅读推进力）| 19/20 | 高 |
| 转发指数（认知 28 + 金句 26）| 54/60 | 高 |
| 长期关注指数（品牌 73 + 案例 20）| 93/120 | 中 |
| **PPT 惩罚** | **-6** | （扣分项）|
| **综合判定** | **3 高 1 中** | **✅ 强推** |

**关键判断逻辑**：
- **流量**是入口，**完读**是过程，**转发**是结果，**长期关注**是品牌
- 4 个指数**互相制约**：流量高但完读低 = 读者点开但划走 = 标题党的反噬
- **PPT 惩罚是减分项**，反映在"完读指数"和"长期关注指数"上
- 不再使用单一总分（避免"总分 80 但 4 维全 60"的伪爆款）

### Step 4：输出格式（严格按这个）


### 输出风格

- 评估要**像资深主编**：敢说"78 分，不是爆款上限"
- 不给模糊分数，每个维度都给具体扣分理由
- **必须修改只能选 1 个最值得改的**——不要列一堆让用户选择困难
- **金句必须全部列出来**——这才是用户能从你这拿走的核心资产
- 转发潜力要给"具体是什么场景"——"转给同事看"还是"发群讨论"必须分清

## 关键原则：结构完整性 ≠ 真实点击潜力

**这是本 skill 最核心的校准**。区分两者：

| 维度 | 结构完整性 | 真实点击潜力 |
|---|---|---|
| 关注 | 标题"写得好不好"（弧线完整/对仗/时序） | 标题"能不能被点开"（热词/身份/1秒理解/具体承诺）|
| 加分项 | "我以为 A，结果 B" 完整弧线 | 命中具体身份词/工具词/场景词 |
| | 完美对仗 | 强动词钩子（淘汰/重做/做错）|
| | 完整时序（半年后/后来才发现） | 短句利落（15-20 字）|
| | 强反转（不是 A 是 B） | 信息承诺明确（具体复盘/数字/判断）|
| 减分项 | 缺弧线/缺时序/缺反转 | 缺热词/缺身份词/抽象词太多/字数超 28 |

**绝对禁止的隐性偏差**：
1. ❌ 不要因为标题有"我以为/后来发现/帮我/时序"就自动给高分
2. ❌ 不要因为标题"结构工整"就判定"高点击"
3. ❌ 不要把"抽象高级"误判成"高点击"
4. ❌ 不要给"金句潜力"加隐藏分——金句在正文里，标题是钩子不是金句
5. ✅ 结构完整性只能是**加分项**（最高 +5），不能替代**热词/身份代入/1秒理解/具体信息承诺**（最高 -20 扣分项）

**典型陷阱**（要记住，下次自动避坑）：

❌ 高结构分低点击的标题（实际案例）：

- **旧版评分："Claude 帮我做产品，我才发现是淘汰错误想法"**
  - 旧版结构分：95/100（"我以为A，结果B"完整弧线 + "在帮我"亲切感 + "后来发现"时序）
  - **新版点击潜力：38/100（偏弱）**
    - 抽象词"淘汰错误想法"：-10
    - 去掉"Claude"后仍成立（泛观点）：-8
    - 关键词太宽泛：-8
    - 缺具体数字/动作锚点
  - **新规则驱动发现了 v18a："Claude Code 半年，92% 都在做错" → 84/100（高潜力）**
    - 用正文真实数字 92% 替代抽象的"淘汰错误想法"
    - 数字反差（92% 错 vs 8% 对）远强于结构完整度

- "Claude Code 用半年我才发现：它做的是产品实验"（结构 96，但"产品实验"是产品圈行话，非产品岗读者 1 秒内不完全理解）

✅ 低结构分高点击的标题：

- "Claude Code 半年，帮我省了 100 个小时"（结构可能 70，但"省 100 小时"是具体数字钩子，点击更高）
- "程序员用 Claude Code 的第 30 天，我辞职了"（结构 75，但"辞职"是强冲突 + "程序员"是身份词，点击远高于结构 96 的标题）

**核心教训**：旧版评分系统性高估结构完整标题的点击潜力。当发现标题"结构分高但没有具体数字/强动词/身份锚点"时，用新规则重新打分——往往比预期低 40-60 分。

**结论**：评"标题是否会被点开"——不要评"标题是否写得工整"。

### 模式 B 陷阱

1. **不要评观点对错**——用户明确说不要。即使你觉得"Demo 价值不是上线"是错的，也不要扣分。
2. **不要混淆模式 A 和 B**——模式 A 看到正文也不评；模式 B 必须评。
3. **不要把"金句多"等同于"内容好"**——案例空、升维空，但堆了 10 个金句，依然是中等偏弱。金句密度只是八维之一。
4. **不要用"高潜力"模糊评级**——必须明确给出"爆款/高/中/偏弱/很弱"之一，并给一句话理由。
5. **不要省略"如果阅读量不高最可能的原因"**——这是用户最想知道的反推逻辑。
6. **必须修改只能给 1 个**——给多了用户就只改一个，干脆只挑 1 个最值改的。
7. **金句数量按"反直觉+短句+视觉感"标准数**——不要把"几小时一个产品"这种"具体细节"也算金句。判断：截图发朋友圈丢不丢脸？丢脸就不算。

## 已知陷阱（模式 A）

1. **外链封面 vision 经常失败**——优先用 file:// 路径 + mcp_browser_vision
2. **"AI 没有替代程序员"这种标题**——目标读者代入感弱（没说"你"也没说"程序员"），属于"看起来在讨论程序员但你不知道是不是写给你的"。要建议加"我们""你"或具体身份词
3. **"X只是把Y重新分层"句式**——认知冲突分高（反转），但"重新分层"是抽象词，扣信息承诺分
4. **封面用纯英文大字**——符合公众号封面规范（用户在 wechat-post 偏好里固化了"纯英文、超大白字、黑色粗描边"），不要扣分
5. **不要给"完整文章评估"**——用户可能想测的是单看标题/封面吸引力，正文好不好是另外的事
6. **副标题在手机信息流几乎不可见**——副标题字号约 12-14px，绝大多数人看不到。主标题必须自己完成"话题+反转"双重任务，不能依赖副标题承载核心冲突。
7. **"AI 时代"开头的标题自动降权**——在程序员/产品经理/AI创业者群体里，"AI 时代 + 大判断"型标题被识别为 AI 观点文，0.3 秒被跳过。保留"AI 时代"必须把它推到"被讽刺/被解构"的位置（如反问句或反转句里）。
8. **直接给结论型标题（如"被严重低估了"）缺问题感**——读者看完不知道"低估了什么/我之前怎么以为的"，认知冲突分为 0。必须改成问题式或反差式结构（"我以为A结果B" / "为什么X但Y" / "不是X，是Y"）。
9. **第一档标题格式（"为什么...但..."）天然高 20+ 分**——"为什么"开头 = 强问题感，"很多 Demo 没人用" = 强冲突 + 身份词落地，"但我还在做" = 强反转。这是基准线，任何低于这个结构的标题都要扣分。

## 全局陷阱

1. **模式 A vs B 选错**——用户只说"打分"时默认走模式 A；用户明确说"全文打分 / 正文打分 / 看完整文章"才走模式 B。模式 A 看到正文不要顺手评，模式 B 评了就要评全套八维。
2. **两种模式不要混用框架**——模式 A 七维是"标题+封面在信息流的吸引力"（不打分内容）；模式 B 八维是"全文传播力"（不评观点对错）。两套分数不能放在同一份报告里。
3. **存档位置**——打分结果存到 `~/.hermes/research/score_YYYYMMDD_HHMMSS.md`（模式 A）或 `score_full_YYYYMMDD_HHMMSS.md`（模式 B），便于复盘历史评分。
4. **迭代使用**——用户说"再打分"/"改一版"时，调取上次打分结果对照，看分数升降。
5. **标题长度预检两种模式都要走**——Step 0 字数扣分表是统一的（15-20 字最佳，过短过长都扣），不挤进七维也不挤进八维，独立显示在"零、标题长度预检"区。**不要再用老的"22 字以上扣 1-2 分"松散规则**。
6. **数字单位一致**——"我以为 X，结果 Y"型标题在 Step 0 算字数时，"我"、"以为"、"结果"都是 1 字；阿拉伯数字"92%"= 1 字；英文单词按 1 字符 = 1 字；英文标题按 1 词 = 1.5 字折算。
7. **标题-正文一致性是隐性大坑**——Step 0.5 预检常被忽略，但"读完觉得被骗了"是低传播率最隐蔽的根因。任何标题"用工具/具体经验"但正文"讲方法论/趋势"的组合都要 -3 起步。改法：把标题视角对齐到正文实际讲的视角，不要试图"两边兼顾"。
8. **拗口是隐性杀手**——Step 0.6 预检的最严重症状是 2 对以上引号 + 主语跳跃 + "X 不如 Y" 双方都是生造短语。这种标题读者需要重读，0.3 秒内直接划走。"X 不如 Y" 结构的 X 和 Y 必须是大众词（快/慢/贵/便宜/稳/玄等）。
9. **结构完整性 ≠ 真实点击潜力**——不要因为标题有"我以为/后来发现/帮我/时序"就给高分。结构分只是加分项（最多 +5），不能替代热词/身份词/1秒理解/具体信息承诺（最高 -20 扣分）。"结构 96 但点击弱" 的典型是"我以为 Claude 在做产品，后来发现是淘汰错误想法"——结构完整但"淘汰错误想法"对很多读者抽象。
10. **去掉"Claude Code" 后还成立的标题是泛观点**——"AI 时代产品方法论彻底变了" 去掉 Claude Code 完全成立 = 行业泛观点（扣 5-15）。好的标题必须"和 Claude Code/具体工具/具体身份强绑定"。
11. **短期流量高 ≠ 标题好（双独立分检查）**——双独立分就是为防止"短期流量自杀长期品牌"。一个标题可能流量分 90 但品牌分 40（数字党/营销号）——直接 ❌ 不推荐。**判定逻辑：双 80+ 强烈推荐 / 流量 80+ 品牌 70+ 可用 / 双 70+ 慎用 / 任一 < 70 不推荐**。
12. **数字有出处是底线**——"92% 都在做错"如果正文有"92% 评估准确率 + 11% 需求命中"的真实数据，加真实性满分；如果是凭空编的"92% 程序员都这样"，真实性 0。"行业数据"和"作者亲历"差很多——前者读者会想"数据从哪来"，后者读者会想"我也这样"。
13. **信任成本 ≠ 真实性，是独立维度**——"92% 都在做错"即使正文有 92% 数据，**读者天然起疑**（"92% 怎么算的？"）→ 信任成本低（4-6 分）。"我以为...结果..." 即使没具体数字，**读者天然信**（"这是作者经验"）→ 信任成本高（12-13 分）。**句式决定信任成本，数字真假决定真实性**。作者读者群（AI/产品/技术）对"百分比+强判断"天然低信任，对"反问/观察/亲历/弧线"天然高信任。
14. **观察式 / 反问式 / 转折式 句式天然高信任**——"大家都在 X，反而 Y" / "我以为...结果..." / "X 越来越多，Y 反而变难" 这类句式天然是"发现"不是"宣告"，读者天然信。这是**最高信任成本**的句式。**陈述式 + 数字 + 强判断** 天然低信任。
15. **双独立分优于单一总分**——设计评分体系时，如果同时评估"流量"和"品牌"，**必须用双独立分而不是单一总分**。单一总分（如 140 分）会导致无法区分"流量 90 品牌 40"和"流量 70 品牌 75"这两种本质不同的情况。双独立分的判定规则：双 80+ 强烈推荐 / 流量 80+ 品牌 70+ 可用 / 双 70+ 慎用 / **任一 < 70 = 不推荐**。这条是本次 skill 重构的核心经验。**任何需要同时评估"短期效果"和"长期影响"的评分体系，都应考虑双独立分架构**。
